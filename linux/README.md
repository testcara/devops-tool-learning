### Linux 发行版本分类与详解（精简面试版）
Linux 仅为系统内核，各类商用/服务器系统都是基于内核封装的发行版。主流分为三大派系 + 容器专用系统，运维云原生岗位重点掌握 Ubuntu、Alpine。

一、RedHat 系（.rpm / yum/dnf）

    企业生产首选，稳定性高，多用于传统服务器业务。
    - RHEL：商业付费企业系统，官方技术支持。
    - CentOS：免费开源，国内经典服务器系统（现已停更）。
    - Rocky/AlmaLinux：CentOS 替代方案，目前主流生产服务器系统。
    - Fedora：社区测试版，软件新、稳定性差，仅用于测试。

二、Debian 系（.deb / apt）

    生态丰富、兼容性好，通用型极强。
    - Debian：稳定保守，是全系基础系统。
    - Ubuntu：基于 Debian，上手简单、生态完善。Ubuntu Server LTS 广泛用于云服务器、Docker/K8s 集群搭建、CI/CD 环境，是云原生运维最常用的服务器系统。
    - Kali：预装渗透工具，用于网络安全测试。

三、SUSE 系

    小众企业系统，多用于政企传统业务，日常运维接触较少。

四、容器专用：Alpine Linux（JD重点）

    极致轻量（镜像仅5MB）、低漏洞、无冗余依赖，专属云原生/容器场景。是 Docker 基础镜像、K8s 微服务镜像的首选系统。

五、核心总结（面试必背）

    - 企业服务器：Rocky、Ubuntu Server、RHEL
    - 容器镜像环境：Alpine
    - 通用开发运维：Ubuntu

六、派系命令区别

    - RedHat系：yum/dnf
    - Debian系：apt
    - Alpine：apk

### Linux 核心网络组件知识点

一、传输层协议

1. TCP
    面向连接、可靠、有序的传输协议，需要三次握手建立连接、四次挥手断开连接，自带重传、校验、拥塞控制机制。开销较高，适用于文件传输、网页访问、数据库通信等对数据完整性要求高的场景。
2. UDP
    无连接、不可靠、无需握手，传输开销极小、速度快，不保证数据有序不丢包。适用于直播、DNS、日志传输、实时通信等追求速度，可容忍少量丢包场景。

二、网络流量管控组件

1. tc（流量控制）
    Linux 内核自带流量管控工具，用于限制网卡带宽、模拟网络延迟、丢包、抖动。云原生场景常用于压测、限制容器出口带宽、排查网络不稳定问题。
2. iptables
    Linux 传统内核防火墙工具，基于 netfilter 框架。可实现数据包过滤、端口转发、SNAT/DNAT、负载均衡。K8s 早期 Service 四层转发底层依赖 iptables，规则链式匹配，适合基础网络策略管控。
3. nft
    iptables 升级版，统一兼容 iptables、arp、ip6 规则，语法更简洁、性能更高、规则可批量管理。目前主流系统逐步替代 iptables，用于新一代服务器、容器网络策略管控。

三、虚拟网络二层组件

1. bridge（网桥）
    Linux 原生二层虚拟交换机，作用和物理交换机一致，可转发二层数据帧。是 Docker 默认网络、K8s 节点基础网络组件，用于连通同一节点内的多个容器。
2. OVS（Open vSwitch）
    开源虚拟交换机，功能强于原生 bridge，支持 VLAN、VXLAN、流量镜像、自定义流表。是 K8s 主流 CNI（Calico/OVN）底层核心组件，多用于复杂集群网络、多租户隔离。

四、网络隔离与隧道组件

1. VLAN
    物理网络二层隔离技术，通过 VLAN 标签划分不同局域网，实现物理交换机下多网段隔离，多用于机房物理网络分区，隔离不同业务流量。
2. VXLAN
    三层网络隧道技术，是K8s 跨节点容器通信核心。通过隧道封装，将二层数据帧封装在三层 UDP 数据包中，实现跨节点、跨网段容器互通，解决传统 VLAN 网络规模限制，适配大规模集群。

五、内核高级网络：BPF
    
    Linux 内核可编程技术，可动态注入内核程序，无需修改内核源码。高性能实现网络过滤、流量统计、安全拦截、网络观测。主流 CNI、容器网络监控、网络安全工具（Cilium）全部基于 BPF 实现，是现代云原生网络核心技术。

六、面试极简总结
    
    - 传输协议：TCP 可靠慢速，UDP 快速不可靠
    - 流量管控：tc 限流，iptables/nft 做转发、防火墙
    - 二层交换：bridge 轻量基础，OVS 功能强大适配集群
    - 网络隧道/隔离：VLAN 物理隔离，VXLAN 跨节点容器隧道通信
    - 高端内核：BPF 支撑高性能云原生网络、观测与安全

### Linux 核心存储组件

    - ext4：简单稳定，Linux 默认单机文件系统，无高级功能
    - ZFS：单机高性能、自带快照纠错，适合本地可靠存储
    - Ceph：分布式高可用，云原生集群持久化存储首选
    - xfs：生产服务器主流，大文件存储最优
    - btrfs：轻量全能Linux文件系统，多用于测试环境
    - tmpfs：内存临时文件系统，高速易丢失
    - NFS：简易跨主机共享存储，适合小型集群

### Linux 常见故障排查工具

1. strace
    系统调用追踪工具。跟踪进程所有的系统调用、参数、返回值。用于排查程序启动失败、权限报错、文件找不到、卡死、静默退出等底层问题。
2. tcpdump
    网络抓包工具。命令行抓包，分析 TCP/UDP 握手、超时、丢包、重传、接口请求异常，是服务器网络排障核心工具。
3. go-pprof
    Go 程序性能剖析工具。专门针对 Go 服务，排查 CPU 暴涨、内存泄漏、Goroutine 泄露、阻塞、函数耗时过高问题，常用于云原生服务性能调优。
4. htop
    进程资源监控工具。升级版 top，可视化查看整机 CPU、内存、各进程资源占用，快速定位占用资源过高的异常进程。
5. iotop
    磁盘IO监控工具。查看每个进程的读写速率、IO 占用，用于排查服务器磁盘 IO 打满、磁盘卡顿、负载过高问题。
6. iftop
    网卡流量监控工具。监控网卡实时上下行流量、TCP 连接流量，用于排查带宽跑满、异常流量、对外请求泛滥等网络问题。
7. dmesg
    内核日志查看工具。打印内核运行日志，专门排查系统级故障：OOM 内存杀死、磁盘损坏、内核报错、驱动异常、服务器突然重启。
8. perf
    Linux 内核级性能分析工具。系统顶级性能排查工具，可定位内核耗时、CPU 热点函数、上下文切换频繁、软中断过高、程序卡死等深层性能瓶颈。
9. cat /proc
    系统内核信息伪文件系统。/proc 不占用磁盘空间，实时展示系统运行态：进程状态、内存、CPU、网络、文件句柄、系统参数，是底层排查问题最基础、最通用的方式。

面试总结（背诵版）

    - 进程异常：strace、htop
    - 网络异常：tcpdump、iftop
    - 磁盘IO瓶颈：iotop
    - 系统内核报错/重启：dmesg、/proc
    - 整机性能瓶颈：perf
    - Go服务性能调优：go-pprof

### Linux 应用层核心软件知识点

1. Nginx

    高性能开源 Web 服务器、反向代理、负载均衡组件。
    核心作用：提供静态资源访问、七层反向代理、流量负载均衡、限流、SSL 卸载、动静分离。是业务网关、K8s Ingress 最主流实现，稳定性高、并发性能强，广泛用于业务流量入口。

2. Postgres（PostgreSQL）

    开源企业级关系型数据库。
    核心作用：存储结构化业务数据，支持事务、ACID 一致性、复杂查询、索引优化、JSON 数据类型。相比 MySQL，扩展性、数据可靠性、复杂查询能力更强，多用于企业核心业务、监控元数据存储。

3. Prometheus

    云原生开源时序监控系统，K8s 官方标配监控组件。
    核心作用：基于 Pull 模式定时采集服务器、容器、服务的指标数据，支持多维度指标、自定义告警规则，持久化时序数据，用于监控服务状态、资源使用率、业务指标、异常告警。

4. Grafana

    开源可视化面板工具。
    核心作用：对接 Prometheus、各类数据源，自定义可视化仪表盘，将监控指标图形化展示，直观呈现 CPU、内存、流量、业务成功率等数据，支撑日常运维观测、故障复盘、性能分析。

5. Fluentd

    开源统一日志采集转发工具，云原生标准日志组件。
    核心作用：跨节点、跨容器采集日志，支持日志过滤、格式化、聚合、转发，可对接 Elasticsearch、对象存储，统一收集容器、服务、系统日志，实现集群日志集中管理。

6. Golang

    编译型静态编程语言，云原生生态首选语言。
    核心特点：编译后二进制可直接运行、无需依赖虚拟机、占用资源低、自带高并发调度、协程轻量化。绝大多数云原生组件（K8s、Docker、Prometheus）均基于 Golang 开发，适合开发高性能微服务、运维工具、网关组件。

7. Rust

    高性能系统级编译型语言。
    核心特点：内存安全、无 GC、极低运行开销、并发安全。多用于底层网络组件、内核插件、容器运行时、高性能中间件开发，适合对性能、稳定性、安全性要求极高的底层服务。

8. Python

    解释型脚本语言，运维通用语言。
    核心特点：语法简单、生态丰富、开发效率高。多用于运维脚本编写、自动化部署、数据处理、监控脚本、简单工具开发，大幅提升运维自动化效率。

面试极简总结

    - 流量网关：Nginx（七层代理、负载均衡）
    - 业务存储：Postgres（企业级关系数据库）
    - 监控体系：Prometheus+Grafana（指标采集+可视化）
    - 日志体系：Fluentd（集群日志统一采集转发）
    - 开发语言：Golang(云原生组件)、Rust(底层高性能)、Python(运维自动化)
